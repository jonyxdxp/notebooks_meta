## What is EBT-Policy?

EBT-Policy is an **implicit visuomotor policy** for robotics. Rather than predicting
actions directly (like a standard regression network), it learns an **energy function**
ddd
```ee
E_θ(o, a)  →  ℝ
```
ddd
that scores the *compatibility* between an observation `o` and a candidate action
trajectory `a`. Lower energy = higher compatibility with expert demonstrations.

At inference time, actions are generated by **minimising this energy** via Langevin
MCMC — the model "thinks" by iteratively refining its action guess.

### Key advantages over Diffusion Policy

| Property | Diffusion Policy | EBT-Policy |
|---|---|---|
| Inference steps | ~100 (fixed) | 2–20 (adaptive) |
| Exposure bias | High | Low (equilibrium dynamics) |
| Uncertainty signal | None | Scalar energy |
| OOD recovery | Fails (diverges) | Emergent retry behaviour |
| Multimodal actions | Yes | Yes |

## Architecture

```
Observation tokens  ─┐
                      ├─► [E | o₁ | … | o_H | a₁ | … | a_K]
Action tokens       ─┘           │
                           Transformer Encoder (bidirectional)
                                  │
                             [ENERGY] token
                                  │
                             Energy MLP  →  scalar E
```

The single scalar `E` is used for:
- **Training**: drive the MCMC trajectory toward ground-truth actions
- **Inference**: select the lowest-energy action from N candidates
- **Uncertainty**: high energy → uncertain → trigger retry

---

## Training objective

```
L = SmoothL1(a_opt, a_gt)          ← trajectory fidelity
  + λ_gp  · E[||∇_a E(o,a)||²]    ← gradient-penalty regulariser
  + λ_reg · E[E(o, a_gt)²]        ← energy magnitude regulariser
```

No contrastive negatives needed — the MCMC unroll itself provides the implicit
contrast between ground-truth and model-generated trajectories.

---

## Langevin sampler

```
Energy-scaled step size:  σ(E) = clip(|E|, σ_min, σ_max)
Nesterov update:          v ← m·v − σ·∇_a E
                          a ← a + v
Langevin noise:           a ← a + √(2σ) · ε,  ε ~ N(0,I)
Early stop:               halt when ||∇_a E||₂ < tol
```

### Novel components vs. prior EBM policies

1. **Energy-scaled MCMC step sizes** — larger steps when the model is uncertain
2. **Pre-sample normalisation** — layer-normalise observations before energy eval
3. **Nesterov-accelerated gradients** — faster convergence in the action space
4. **Randomised training steps** — stochastic exploration of the energy landscape
5. **Adaptive early stopping** — 50× compute reduction on easy inputs

## Using the library in your own code

```python
from ebt_policy import EBTPolicyConfig, EBTPolicyTrainer

cfg = EBTPolicyConfig(
    obs_dim        = 64,   # e.g., flattened image features
    action_dim     = 6,    # 6-DoF end-effector delta
    obs_horizon    = 2,
    action_horizon = 16,
    d_model        = 256,
    n_layers       = 6,
    n_epochs       = 100,
)

trainer = EBTPolicyTrainer(cfg, device="cuda")
trainer.fit(obs_data, action_data)   # Tensors of shape (N, H_o, d_obs) and (N, H_a, d_a)

# Inference
obs_t = ...   # (1, H_o, d_obs)
action, energy = trainer.sampler.infer(trainer.model, obs_t)
```

### Uncertainty-aware retry controller

```python
from ebt_policy import UncertaintyAwareController

ctrl = UncertaintyAwareController(trainer, retry_threshold=0.3, max_retries=3)
action, energy, n_retries = ctrl.act(obs_t)

if n_retries > 0:
    print(f"Recovered after {n_retries} retries (energy={energy:.3f})")
```

## Emergent retry behaviour

The paper's headline result: EBT-Policy learns to **retry failed actions** from
pure behaviour cloning, without any explicit retry training data.

The mechanism: when the energy after execution is high (contact failure), the
controller re-samples with a wider noise distribution, automatically exploring
nearby action candidates. This emerges from the equilibrium dynamics of the
energy function — not from any supervised signal.
